# Grid レイアウトを使ったページレイアウト

Web デザインの世界では、近年 Grid レイアウトという新しい CSS のレイアウト手法が登場しました。
この手法は柔軟性のあるレイアウトが可能な点が特徴で、Web デザイナーやフロントエンドエンジニアにとって、Web のデザインを向上させる有力な手段となっています。

Grid レイアウトの仕様である CSS Grid Layout Module Level 1 は 2016 年に Candidate Recommendation となり、およそ 2017 年に主要なブラウザでの実装が完了しました。
Vivliostyle では長らく Grid レイアウトに対応していませんでしたが、Vivliostyle.js v2.17.0（2022 年 7 月 29 日リリース）よりモダンブラウザの大部分の CSS プロパティがサポートされ、Grid レイアウトも使用できるようになりました。
Web デザインにおいて長年求められてきた機能だけに、非常に強力なレイアウト手法であるため、組版においても活用できる点は多いです。
このドキュメントでは、Grid レイアウトを使用する際の CSS 組版ならではの考慮点について紹介します。

## Grid レイアウトの概要

Grid レイアウトはその名の通り、グリッドと呼ばれる縦方向と横方向で区切られた領域をベースとしたレイアウト手法です。
Grid レイアウトは、要素に対して `display: grid` を設定することで適用します。

```html
<div class="grid">
  <div class="container-1">Container 1</div>
  <div class="container-2">Container 2</div>
  <div class="container-3">Container 3</div>
  <div class="container-4">Container 4</div>
  <div class="container-5">Container 5</div>
</div>
```

```css
.grid {
  display: grid;
}
```

これで `.grid` 要素がグリッドコンテナとなり、子要素がグリッドに沿って配置されるようになります。
さらに、`grid-template-columns` や `grid-template-rows` プロパティなどを使って、グリッドの行と列を定義します。

グリッドコンテナの子要素は、デフォルトでは左上から順にグリッドに配置されていきますが、配置される位置も制御できます。
`grid-column` や `grid-row` プロパティを使って、グリッド内の各アイテムの位置を調整します。
さらに、グリッドを縦方向や横方向に結合させたり、グリッド間に Gap と呼ばれる隙間を設置することも可能です。

## CSS 組版における Grid レイアウト

紙面をレイアウトする CSS 組版は、Web とは異なりスクロールという概念がないため、縦方向（縦書きなら横方向）に物理的なサイズの制限があります。
このとき、一つのグリッドコンテナを複数ページにまたがって扱おうとするとページ送りの処理で意図しないレイアウトになることが多いため、基本的には 1 ページを一つのグリッドコンテナとみなすと直感的に扱えるかと思います。
Vivliostyle では、`100vw` をページの横幅、`100vh` をページの縦幅として解釈してくれるため、以下のようにすると（上下左右の余白を除いた）ページの領域全体にグリッドコンテナが配置されます。

```css
.grid {
  display: grid;
  width: 100vw;
  height: 100vh;
}
```

![紙面上のグリッドコンテナのイメージ図](/pages/articles/grid-system/fig-1.svg)

ページを縦方向や横方向に分割できる機能はあたかも段組レイアウトのようですが、グリッドの領域はお互いに重ね合わせることができても、グリッドの領域を四角形以外に形作ることができません。
そのため、いわゆる文章の流し込み（複数の段組みやページに沿って文章を配置すること）などは Grid レイアウトを使っても表現することはできません。
Grid レイアウトは、例えば写真集のように、画像や短い文章など一つのグリッド領域で要素が完結するものを柔軟に並べ替えたいときに力を発揮するでしょう。

# spring-raining 式 Grid System

以上の点を考慮した上で、Vivliostyle で Grid レイアウトを扱いやすくした Vivliostyle Theme **vs-theme-grid-system** を作成してみました。

テーマをインストールするには `vivliostyle.config.js` に以下のように設定します。

```js
module.exports = {
  theme: [
    '@spring-raining/vs-theme-grid-system@1',
    // 他に使用するテーマファイルがあればここに記述
  ],
};
```

テーマをインストールすると、class 名に `grid-page` を指定した要素が 1 ページ全体のグリッドコンテナになります。
そして、このグリッドコンテナ以下にそのページに配置したい要素を追加していくことでレイアウトしていきます。
それぞれの要素は `grid-column` や `grid-row` を設定しても良いですし、`--vs-theme--page-grid-area` という CSS 変数が用意されているのでそれを使って配置しても良いです。

```html
<div class="grid-page">
  <div style="--vs-theme--page-grid-area: 1 / 2">下にネコの画像を配置</div>
  <img
    src="./cat.png"
    alt="ネコの画像"
    style="--vs-theme--page-grid-area: 2 / 2"
  />
</div>
```

または、以下のように CSS 変数を使って指定することも可能です。

```css
:root {
  --vs-theme--page-grid-inner-template-cols: repeat(2, 1fr);
  --vs-theme--page-grid-inner-template-rows: repeat(3, 1fr);

```

そのため、以下のようにキーワードを使って直感的に領域を指定できます。

```html
<div class="grid-page">
  <!-- 以下の2つは同じ場所に配置される -->
  <div style="--vs-theme--page-grid-area: 2 / 2 / 3 / 3"></div>
  <div style="--vs-theme--page-grid-area: page-inner"></div>
  <!-- 以下の2つは同じ場所に配置される -->
  <div style="--vs-theme--page-grid-area: 1 / 1 / 4 / 4"></div>
  <div style="--vs-theme--page-grid-area: page-outer"></div>
</div>
```

## 見開きページへの対応

Vivliostyle は、これまで他の組版システムにあるような見開きページ（左右 2 ページが連続したレイアウト）への対応を苦手としてきました。
今回作成した Grid System は、実験的にこの見開きページに対応するための設定を追加しています。

例えば左右 2 ページ全体に一枚の画像を表示させたい場合、以下のようにします。

```html
<div class="facing-pages">
  <div class="grid-page">
    <img
      src="image.png"
      alt="見開き画像"
      style="--vs-theme--page-grid-area: page-outer"
    />
  </div>
  <div class="grid-page">
    <img
      src="image.png"
      alt="見開き画像"
      style="--vs-theme--page-grid-area: page-outer"
    />
  </div>
</div>
```

まず、クラス名 `facing-pages` の div 要素を配置し、その中に `grid-page` 要素を**2 つ**用意します。
HTML 上では画像が 2 つ用意されていますが、Vivliostyle で表示するとちゃんと 2 ページに分割された見開きページになっているかと思います。

ではなぜ同じページを 2 つ用意するのでしょうか？
これは、Vivliosyle のレイアウト処理に関係しています。
Vivliostyle では、組版処理の過程で HTML 要素を特定の 1 ページに割り当てていきます。
そのため、HTML 要素が 1 つだけの場合、どうしても左右いずれかのページにしか表示されなくなります。
この問題を回避するため、Grid System では全く同じページを 2 つ用意しておきます。
そして、片方のページは左半分のみ、もう片方のページは右半分のみ表示させる、というレイアウトにより見た目上は 1 つの見開きページとなるようにしています。

![見開きページレイアウトのイメージ図](/pages/articles/grid-system/fig-2.svg)

ちなみに、Vivliostyle 以外では 1 つ目の `grid-page` 以外は非可視化されているので、通常の画面表示でも問題なく表示されます。
